Product surface

核心卖点是伙伴和聊天体验。

Mia 不要求用户记住每个 CLI 怎么启动。你看到的是伙伴、对话、状态和权限；底层内核留在该在的位置。

Multiple fellows 多个 AI 伙伴

不同 Fellow 可以有不同身份、头像、技能和模型配置，不再只有一个泛化助手。

Agent cores 兼容多 Agent 内核

Hermes 提供开箱体验；Claude Code 和 Codex 继续复用你本机的账号、PATH 和文件系统。

Multi-device 桌面端和 Web 互通

Web 端可以进入对话；需要本机工具时，通过在线桌面 bridge 回到 owner desktop。

Chat UX 舒服的前端 GUI

会话列表、群聊、附件、权限卡片、工具进度和回复都围绕聊天体验组织。

Trust boundary

兼容很多内核，但执行边界要清楚。

Hermes packaged. Claude Code and Codex stay yours. 内置 Hermes 负责默认体验；Claude Code / Codex 仍由你的本机环境和账号控制。
权限先问，再执行 用户能看到任务计划、命令意图和执行位置，而不是只看流式回复。
多端互通，不等于云端接管 远程请求只有在 owner desktop 在线并绑定时才会进入本机执行路径。
