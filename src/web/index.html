Local-first agent control

Claude Code 和 Codex 保持在你的 PATH、账号和文件系统里运行。云端可以传递请求，但不能伪装成本机。

$ mia route @kongling --engine codex
runtime: owner-desktop
permission: ask-before-shell
reply: cloud-room-message

Agents Hermes / Codex / Claude Code

信任边界要看得见。

Hermes packaged. Claude Code and Codex stay yours. 内置运行时负责可控的默认体验；你自己的 CLI Agent 仍由本机环境和账号控制。
权限先问，再执行 敏感动作不被藏进流式回复里，用户能看到任务计划、命令意图和执行位置。
Bridge 在线才调用你的 Fellow 远程请求只有在 owner desktop 在线并绑定时才会进入本机执行路径。
